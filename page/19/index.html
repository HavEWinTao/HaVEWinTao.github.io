146.LRU 缓存

题目描述

模拟一个cache的LRU(最近最少使用)的调度策略

其中cache的容量是capacity

解题思路

其实体系结构操作系统课都学过LRU的原理，怎么用编程来实现呢？

体系结构课给出的描述是：首先应该有一个栈，这个栈可以将一个元素给调到栈顶，然后还可以将超出容量的元素从另一端挤出去(应该是这样)

实现方式：

双向链表+Map

双向链表的结点保存了一个value，然后在Map中可以根据key找到这个双向链表中的结点

这样的话就是在头部插入元素，在尾部删除元素，将某一元素调整到头部三个操作

就是一共有numCourses门课，然后有一些先修课程的要求，判断能够完成所有的课程学习

解题思路

其实一共有numCourses个结点，然后先修课程的关系构成一张图

先修课程对[0,1]表示：想要学习课程0，你需要先完成课程1，也就是有一个结点1指向0的边

判断这个图中是否存在圈

每个节点有三个状态

  1. 未搜索：还没有搜索到这个节点

  2. 搜索中：我们搜索过这个节点，但还没有回溯到该节点，即该节点还没有入栈，还有相邻的节点没有搜索完成）；

  3. 已完成：我们搜索过并且回溯过这个节点，即该节点已经入栈，并且所有该节点的相邻节点都出现在栈的更底部的位置，满足拓扑排序的要求。

954.二倍数对数组

题目描述

给定一个长度为偶数的整数数组arr，只有对arr进行重组后arr[2*i+1]=2*arr[2*i]时，返回true；否则，返回false

解题思路

肯定要先排序，然后记录每个元素的个数

直接用TreeSet就可以了，TreeMap会将key排序，并且记录每个数字的数量

然后对key进行判断，key>0的话就是找key*2

key<0的话就是先判断key是否能被2整除，再找key/2

每次匹配后都得把这些数减去相应的数量，当找不到匹配的数时就返回false，不然就是true

287.寻找重复数

题目描述

给定一个包含n+1个整数的数组nums，其数字都在[1,n]范围内（包括1和n），假设 nums 只有 一个重复的整数，返回这个重复的数。

要求:不修改原数组,额外空间O(1),时间复杂度O(n)

解题思路

做一个i到nums[i]的映射

如[1,3,4,2,2]就是0-->1-->3-->2-->4-->2 将(i,nums[i])抽象成了一个链表,最后的4-->2就构成了一个环

这个题就和142.环形链表II一模一样了

142.环形链表II

发表于 2022-03-31 更新于 2022-09-18 分类于 LeetCode刷题

142.环形链表II

题目描述

给定一个链表的头节点head，返回链表开始入环的第一个节点。如果链表无环，则返回null。

解题思路

141.环形链表是判断链表是否存在环，该题是找到环的入口

如果不存在环，那么快指针一定会走到null

如果存在环，设链表的左侧非环长度是m，环形长度是c

快指针走了2n个结点，慢指针走了n个结点，那么慢指针就在环内走了n-m个结点

让慢指针再在环内走m个结点那么他就来到了环的起始位置，这样的话可以另取一指针指向头部，这个指针走过m个结点时正好和慢指针在环的起始位置相遇，作为答案返回即可

1606.找到处理最多请求的服务器

发表于 2022-03-30 更新于 2022-09-18 分类于 LeetCode刷题

1606.找到处理最多请求的服务器

题目描述

k个服务器，一个服务器同一时间只能处理一个请求

一个arrive[]请求到达时间数组，一个load[]请求工作时间数组

第i个请求优先分配给第i%k个服务器，如果该服务器繁忙则继续查找下一个服务器，这些服务器视作一个环

返回最繁忙的服务器列表

解题思路

用一个数组记录每个服务器的执行任务的数量

用一个TreeSet来记录可用的服务器isAvailable

用一个优先队列isBusy记录任务完成时间以及这个服务器序号，按完成时间升序排序

开始模拟任务达到的过程

1. 记录到达时间
2. 如果有已经执行完任务的服务器将他从isBusy中弹出并且加入到isAvailable中去
3. 如果isAvailable为空这个任务执行不了
4. 否则按规则找到可以执行的服务器
5. 更新isAvailable和isBusy

最后遍历每个服务器执行任务的数量并得出答案
