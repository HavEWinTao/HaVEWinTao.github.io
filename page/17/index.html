388.文件的最长绝对路径

题目描述

文件系统中的每个文件和文件夹都有一个唯一的绝对路径，所有路径用'/'连接

给定一个以上述格式表示文件系统的字符串input，返回文件系统中指向文件的最长绝对路径的长度

如果系统中没有文件，返回0

解题思路

需要建立一个树状结构

需要判断这个结点是不是文件夹，还是文件，其次需要保存当前节点的所有子节点，rank是路径等级，path是当前节点名

想求最长路径呢，其实就是个DFS的过程，但是显然本题不需要建立完整的树形结构

用一个栈，栈顶是当前的路径，然后不断的向里面加该文件夹的文件和文件夹，文件夹入栈

其中需要涉及到回溯的问题，比如不断更深层的文件夹查找，查找到底的话会返回上一级

其中根据'的数量来判断当前文件的层级，这个层级在回溯时需要用到，回溯就是弹出栈顶的元素

遇到文件就求当前路径长度并和ans对比，最后返回ans

6072.转角路径的乘积中最多能有几个尾随零

题目描述

解题思路

这个题即使先将给定的数组按2和5因子过滤一遍，比如13就变成1，14就变成2这样，然后再去求乘法，也会溢出

转角路径有六种情况： 1. 横着的 2. 竖着的 3. 左到右，然后向上 4. 左到右，然后向下 5. 右到左，然后向上 6. 右到左，然后向下

因为是求尾随0的个数，所以直接将整排整列乘起来就行了，因为多乘一个数也不会影响结果

借鉴238.除自身以外数组的乘积的思路

需要先构建从左到右，从右到左，从上到下，从下到上的乘积

对于(i,j),想求他的转角路径，比如左到右然后向上，就是left2right[i][j]*down2up[i][j]*grid[i][j]

然后求最大的尾随零的个数

但是这样会导致溢出，解决办法如下：

用facter[][][]记录2和5的因子的个数，同时left2right也不算乘积了，也直接记录2和5因子的个数

```Math.min(down2up[i][j][1] + left2right[i][j][1] + facter[i][j][1],down2up[i][j][0] + left2right[i][j][0] + facter[i][j][0]);```

然后求6种转角路径的最多尾随0的个数

时间复杂度为O(mn)

  • 遍历一次找最高值-
  • 从左开始遍历记录左侧最高值同时更新答案，直到遇到最高值，记录这个index1，再从右往左遍历记录右侧最高值同时更新ans，直到遇到最高值，记录这个index2，然后更新index1-index2之间的ans

309.最佳买卖股票时机含冷冻期

题目描述

与121.买卖股票的最佳时期类似

  • 可以多次交易
  • 卖出后第二天不能买入
  • 再次买入前必须卖出之前的股票

解题思路

dp[n][3]，dp表示累计最大收益

dp[i][0]表示目前持有一支股票

dp[i][0] = max(dp[i-1][0],dp[i-1][2]-prices[i])

前一天就有股票，或者不持有然后今天买了股票

dp[i][1]表示目前不持有任何股票，并且处于冷冻期中

dp[i][1] = dp[i-1][0]+prices[i]

前一天还持有股票，然后今天卖了

dp[i][2]表示目前不持有任何股票，并且不处于冷冻期中

dp[i][2] = max(dp[i-1][1],dp[i-1][2])

第i天结束时冷却期也结束了，说明是第i-1天卖的，或者i-1天不持有

初始条件为dp[0][0]=-prices[0]
