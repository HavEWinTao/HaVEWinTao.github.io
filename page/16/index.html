440.字典序的第K小数字

题目描述

给定n,k，返回[1,n]中字典序第k小的数组

思路

一定要用long,不然会溢出导致结果错误

  1. 类似于386.字典序排数，使用一个字典树保存各个数字，然后再先序遍历，但是针对本题的数据量建立字典树内存不够

  2. 然后写了个O(n)的算法

    其实也是想按字典树来写的，也不怎么就写成了个O(n)的，同时因为用了int导致d了好长时间bug，最后发现是long的问题，改成long后答案正确但是会TLE

  3. 模拟字典树

    首先应计算子树共有多少结点

    然后根据子树结点的个数来判断下一步操作，是右移还是下移

    通过不断地右移下移就可以找到第k个数字

MySQL安装与配置

MySQL安装

Windows

  1. 下载官网的.zip包,将其解压到一个路径

  2. 去bin文件夹下执行mysqld --install

  3. 执行mysqld --initialize --console会生成一个密码

  4. 启动mysql服务net start MySQL

  5. 登录数据库mysql -u root -p,密码就是上面生成的随机密码

  6. 修改密码alter user 'root'@'localhost' identified by '123456';(记得use mysql选择数据)

记得将mysql\bin放入到环境变量中去

这种安装和直接的.msi安装相比可以自定义安装目录，并且connector也不是预置的，使用时需要额外的下载

services.msc可以启动服务

linux

  1. 访问https://dev.mysql.com/downloads/repo/apt/获取配置包下载地址

  2. wget https://dev.mysql.com/get/mysql-apt-config_0.8.14-1_all.deb下载MySQL APT配置包

  3. sudo dpkg -i mysql-apt-config_0.8.14-1_all.deb

  4. sudo apt-get update

  5. sudo apt-get install mysql-server安装mysql

通过以上的APT方式安装好之后，所有的服务、环境变量都会启动和配置好，无须手动配置

服务管理

查看服务状态sudo service mysql status

停止服务sudo service mysql stop

启动服务sudo service mysql start

重启服务sudo service mysql restart

相关目录

数据库目录：/var/lib/mysql/

配置文件：/etc/mysql(如my.cnf)

相关命令：/usr/bin(mysqladmin、mysqldump等命令)

启动脚本：/etc/init.d/mysql

卸载

sudo apt purge mysql-* sudo rm -rf /etc/mysql/ /var/lib/mysql sudo apt autoremove sudo apt autoclean

141.环形链表

题目描述

给定一个链表，判断链表中是否有环

解题思路

1.哈希表:将已经出现的结点放入哈希表中，判断是否出现过

2.取巧，因为题目中说最大结点数是10000，所以执行10000次，如果结点还没等于null就说明存在换

3.方法2的通用办法——快慢指针

一个指针取next另一个指针取next.next，然后遍历，如果存在环一定会存在量指针相同的情况，不然就会遍历到尾部判断出不存在环

160.相交链表

题目描述

给定两个链表，判断两个链表是否相交

解题思路

1.用一个set保存结点，再判断集合中是否存在结点来判断相交

2.双指针

一个指针pa指向链表A，另一个指针pb指向链表B，假设链表A比链表B长

pb已经遍历完时pa还没有遍历完，令pb指向链表A的头部，然后继续遍历
